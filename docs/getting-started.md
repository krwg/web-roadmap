# Начало работы

## Сколько времени

| Параметр | Значение |
|----------|----------|
| Длительность | **22 недели** (154 дня) |
| Нагрузка | **6–7 часов в день** |
| Итог | Junior full-stack portfolio + понимание стека |

Если у вас меньше времени — растяните недели; не пропускайте **проекты недели**.

## Репозиторий обучения

Создайте публичный репозиторий, например `my-web-roadmap-journey`:

```
my-web-roadmap-journey/
├── week-01/          # HTML
├── week-02/          # CSS
├── ...
├── week-22-capstone/ # финальный проект
└── journal.md        # дневник: что узнал, что сломалось
```

**Правило:** минимум один осмысленный коммит в день.

```bash
git commit -m "week 3 day 5: responsive gallery with grid"
```

## Распорядок дня (6–7 часов)

Pomodoro: **50 мин работа / 10 мин отдых**.

| Блок | Время | Содержание |
|------|-------|------------|
| 1. Теория | 1.5 ч | Документация, конспект своими словами |
| 2. Практика | 2.5 ч | Задания дня **без копипаста из ИИ** |
| 3. Ревью | 1 ч | Дебаг, чеклист, ИИ только для ревью |
| 4. Reverse engineering | 1 ч | Разбор чужого кода с GitHub |

---

## Протокол «Анти-вайбкодер»

ИИ **не пишет код за вас** на этапе обучения. Допустимо:

1. *«Объясни Event Loop аналогией, без кода»*
2. *«Найди проблемы в моём коде. Не переписывай — укажи строки»*
3. *«Какие edge cases я не учёл?»*

---

## Как не бросить

1. **Тупой час** — не понял тему, не идёшь дальше; рисуй схемы, микро-примеры
2. **20 минут** — застрял → прогулка, вода, без скролла
3. **Коммит каждый день** — даже маленький
4. **Дневник багов:** Симптом → Причина → Решение
5. **Проекты недели обязательны** — они склеивают теорию в навык
