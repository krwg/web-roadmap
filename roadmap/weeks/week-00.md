# Неделя 0: Онбординг — до первого HTML

> **Цель:** подготовить инструменты, аккаунты и репозиторий `learning-log` до [недели 1](weeks/week-01.md).
> **Время:** 2–4 часа (один раз)
> **Проект:** инфраструктура обучения, не код недели

## Зачем отдельная неделя 0

Неделя 1 сразу про HTML **и** Git. Если VS Code, Git и GitHub не готовы — первый день уйдёт на установку, а не на `index.html`. Неделя 0 снимает трение.

---

## День 0.1: Инструменты
<a id="week-00-day-0"></a>

> **Время (полный):** ~1.5ч теория · ~2.5ч практика · ~15м Git · ~1ч ревью  
> **Время (лайт):** ~45м теория · ~1.5ч практика (MVP) · ~15м Git  
> **Связь с проектом:** шаг к **проект недели**

### Теория

Веб-разработка начинается не с кода, а с **среды**: редактор, браузер с DevTools, терминал, Git. Эти инструменты останутся с вами на годы — потратьте время на нормальную настройку один раз.

**Ключевая мысль:** стабильная среда важнее скорости «сразу писать код».

**Читать:**
- [docs/getting-started.md](../../docs/getting-started.md)

### Практика
1. Установи **VS Code**, **Chrome**, **Git**
2. Расширения: Live Server, GitLens, ESLint, Prettier, Error Lens
3. Проверь: `git --version`, `node --version` (Node с нед. 11, можно позже)
4. Открой DevTools (F12) → вкладки Elements, Console, Network

## День 0.2: GitHub и learning-log
<a id="week-00-day-0"></a>

> **Время (полный):** ~1.5ч теория · ~2.5ч практика · ~15м Git · ~1ч ревью  
> **Время (лайт):** ~45м теория · ~1.5ч практика (MVP) · ~15м Git  
> **Связь с проектом:** шаг к **проект недели**

### Теория

Публичный `learning-log` — ваше **портфолио в процессе**. Рекрутер видит не только финальные проекты, но и дисциплину: коммиты, сообщения, рост.

**Ключевая мысль:** репозиторий создаётся до первой строки HTML.

### Практика
1. Аккаунт GitHub (если нет)
2. Новый репозиторий `learning-log` (public, без README)
3. `git clone`, `.gitignore`, первый коммит
4. Скопируй [README-шаблон](https://github.com/krwg/web-roadmap) из прошлого чата или создай минимальный README со ссылкой на [web-roadmap](https://github.com/krwg/web-roadmap)

## День 0.3: Как учиться по маршруту
<a id="week-00-day-0"></a>

> **Время (полный):** ~1.5ч теория · ~2.5ч практика · ~15м Git · ~1ч ревью  
> **Время (лайт):** ~45м теория · ~1.5ч практика (MVP) · ~15м Git  
> **Связь с проектом:** шаг к **проект недели**

### Теория

Каждый день недель 1–22:
- **Теория** — проза + ссылки (не пропускай)
- **Практика** — руками, без копипаста из ИИ
- **Git** — минимум один коммит
- **Лайт-режим** — если мало времени: сократи практику до MVP, теорию и Git оставь

Протокол «анти-вайбкодер»: ИИ объясняет, не пишет за вас.

**Читать:**
- [roadmap/introduction.md](../introduction.md)
- [docs/projects.md](../../docs/projects.md)

### Практика
1. Прочитай introduction и каталог 22 проектов
2. Создай `journal.md` в корне `learning-log`
3. Запиши цель: зачем вам full-stack через 22 недели
4. Открой [krwg.github.io/web-roadmap](https://krwg.github.io/web-roadmap/) — закладка на каждый день

### Если застрял

Застрял >20 мин — остановись. Нарисуй схему на бумаге, сделай минимальный пример в отдельном файле `playground.*`, не копируй готовое решение. ИИ — только объяснить концепцию без кода.
