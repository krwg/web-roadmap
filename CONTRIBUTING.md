# Contributing to web-roadmap

Спасибо, что хотите улучшить открытый учебный маршрут. Любой вклад ценен: опечатки, ссылки, задания, переводы.

## Что можно улучшать

- Исправления в теории и формулировках
- Актуальные ссылки (битые URL, устаревшие API)
- Новые практические задания и edge cases
- Ловушки и ревью-вопросы
- Перевод README / недель на английский (в отдельных файлах)
- Диаграммы и схемы

## Чего мы не принимаем

- Копипаст курсов и книг без разрешения
- Промо платных курсов без явной пользы для ученика
- Упрощение заданий до «скопируй код из ChatGPT»

## Workflow

1. **Fork** репозитория
2. Создайте ветку: `fix/week-05-typo`, `feat/week-12-extra-exercise`
3. Внесите изменения в `roadmap/weeks/` или `docs/`
4. Проверьте Markdown локально
5. Откройте **Pull Request** с описанием:
   - что изменено
   - зачем (какая проблема ученика решается)
   - неделя/день, если применимо

## Стиль

- Язык недель: **русский**, технические термины на английском где принято (useState, middleware)
- Ссылки: предпочтительно MDN, официальная документация, learn.javascript.ru
- Практика: нумерованные шаги + чеклист `- [ ]` в конце
- Один PR — одна логическая тема (не смешивайте неделю 3 и 18)

## Issues

Перед крупным изменением откройте **Issue**: опишите проблему или предложение. Для мелких правок можно сразу PR.

Шаблоны:

- 🐛 Ошибка в задании / ссылке
- 💡 Предложение по содержанию
- 📚 Новый ресурс в библиотеку

## Code of Conduct

Участвуя, вы соглашаетесь с [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
